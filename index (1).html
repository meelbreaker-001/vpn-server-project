Internship Project · WireGuard + Python

A VPN server you can read, test, and reason about end to end.

vpnserver is a Python control plane around WireGuard — it generates keys, allocates addresses, opens the tunnel, and exposes the whole thing through a small REST API. The crypto is WireGuard's. The orchestration is yours.

How it works

Three layers, each doing one job

The project deliberately doesn't reinvent cryptography — it wraps a reviewed implementation in a control layer that's small enough to actually understand.

01 · tunnel

WireGuard

Handles key exchange, encryption, and packet routing. Battle-tested, ~4,000 lines, nothing custom.

02 · control

core/

Thin Python wrappers around wg and iptables — the only layer that touches root.

03 · application

peers/ + api/

Pure Python peer storage, IP allocation, and a FastAPI REST surface. Fully unit-testable, no root needed.

Architecture

What happens when you add a peer

One request triggers a short, traceable chain — no hidden state.

POST /peers
Client requests a new peer by name via the REST API.
keys.py
Generates a fresh Curve25519 keypair and preshared key.
manager.py
Allocates the next free IP from 10.8.0.0/24 and persists the peer record.
wireguard.py
Hot-adds the peer to the running wg0 interface — no restart needed.
qrcode_gen.py
Returns a config file and QR code the client scans into the WireGuard app.
Quick start

Running locally

Clone, install, and bring the API up in under a minute.

# clone and enter the project
git clone https://github.com/your-username/vpn-server-project.git
cd vpn-server-project

# set up environment
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env

# run the API
uvicorn vpnserver.api.app:app --host 0.0.0.0 --port 8000
